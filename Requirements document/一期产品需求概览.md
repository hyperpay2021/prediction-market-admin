# 一期产品需求拆分

版本：v0.3

更新时间：2026-06-04

## 0. 一期核心决策

- 产品形态：标的聚合 + 单标的绑定平台交易 + Web3 非托管钱包 + 快捷/专业双交易面板。
- 接入平台：Polymarket、Predict.fun。
- 交易范围：Maker + Taker 都进入一期。
- 展示逻辑：同一主题可归组，但不合并盘口和深度。
- 执行逻辑：用户选中哪个标的，就把订单提交到该标的绑定的平台。
- 账户逻辑：EVM 地址作为用户唯一 ID；平台不托管资产、不代用户开设上手账户、不做 managed balance。
- 网络逻辑：Polymarket 标的使用 Polygon，Predict.fun 标的使用 BSC/BNB Chain。
- UI 重点：快捷交易面板是核心竞争力，专业交易面板服务熟练用户。
- 自动报单：用 AI 筛选热门、高流动性、赔率清晰的标的，生成快捷购买入口。

## 1. 一期目标

### 1.1 产品目标

让用户在一个平台里发现并交易 Polymarket 和 Predict.fun 的优质预测市场标的：

- 不需要分别打开两个平台寻找热门机会。
- 不需要理解平台品牌和链路差异。
- 不需要把资产托管到我们平台。
- 可以用自己的 EVM 钱包完成授权、签名、下单、撤单和查询。
- 小白用户可以通过快捷面板理解“花多少钱、买什么结果、最多亏多少、成功后得到多少”。
- 专业用户可以通过专业面板查看盘口、设置限价、管理挂单。

### 1.2 业务目标

- 在 7 月底前上线可真实交易或白名单灰度的初版。
- 完成 Polymarket + Predict.fun 的标的采集、展示、快捷交易和专业交易闭环。
- 形成后续二期盘口合并、跨平台路由、自研市场、做市商能力的数据基础。

### 1.3 成功标准

- 用户可以浏览 Polymarket 和 Predict.fun 的精选标的。
- 同一主题的多个平台标的可以被归组展示；如果投资参数、赔率或结算边界不同，前台保持多个独立交易入口。
- 如果同一主题在两个平台的核心投资参数完全一致，前台可以采用“合并主题 + 平台/标的切换”的展示方式，但用户下单前必须明确选中一个具体平台标的。
- 不论展示上是否合并主题，交易层仍保持一笔聚合订单只映射一个底层平台订单，不合并盘口、不合并深度、不共享成本池。
- 当同主题标的出现跨平台价格组合机会，例如买入最优 Yes 与买入最优 No 的总成本小于 1.00 美元，且扣除手续费、滑点和链上成本后仍可能成立时，可以在标的卡片或详情页展示“套利空间”提示。
- 用户可以用 EVM 钱包连接平台，钱包地址成为用户 ID。
- 用户在 Polymarket 标的下单前自动切 Polygon，在 Predict.fun 标的下单前自动切 BSC。
- 用户可以完成 Taker 买入，也可以创建和撤销 Maker 限价单。
- AI 自动报单 banner 能引导用户完成一次快捷下单。
- 订单、成交、持仓可以通过底层平台 API 和钱包地址聚合展示。
- 风控命中时不会提交真实订单。

## 2. 用户与角色

### 2.1 小白用户

- 主要使用移动端或简化页面。
- 关注推荐标的、概率、投入金额、最大亏损、成功后收益。
- 不希望看到复杂平台、链、订单簿和签名细节。
- 更适合使用快捷交易面板。

### 2.2 专业用户

- 主要使用 PC。
- 关注盘口、价格、数量、成交历史、限价挂单、撤单、持仓。
- 可以接受在详情页或订单详情中看到底层平台、链、订单 hash。
- 更适合使用专业交易面板。

界面模式要求：

- 移动端和 PC 端都必须提供快捷版与专业版两套交易界面，不能把快捷版只做成移动端能力，也不能把专业版只做成 PC 能力。
- 默认模式可以按场景区分：新用户、移动端、AI 自动报单入口默认进入快捷版；熟练用户、PC 端、盘口入口可以默认进入专业版。
- 用户选择过的界面模式需要在同一钱包地址或本地设备上记忆，后续进入同类页面时沿用其偏好。
- 两套界面切换时，必须保留用户已输入的方向、金额、价格等核心字段，避免切换导致下单意图丢失。

### 2.3 运营/审核员

- 审核事件主题归组。
- 审核 AI 自动报单文案和推荐优先级。
- 处理低置信度或规则复杂标的。
- 暂停异常标的、异常平台或异常推荐。

### 2.4 系统服务

- 市场数据同步 Worker。
- 标的标准化服务。
- 事件主题归组服务。
- AI 自动报单服务。
- 钱包与网络适配服务。
- 单平台交易执行服务。
- 订单/持仓同步服务。
- 风控与审计服务。

## 3. 一期范围

### 3.1 P0 必须交付

#### 3.1.1 市场与标的数据接入

能力：

- 定时同步 Polymarket 事件、市场、outcome、价格、流动性、成交量、结束时间、状态、结算规则链接。
- 定时同步 Predict.fun 市场、outcome、价格、流动性、成交量、结束时间、状态、结算信息。
- 建立原始平台市场表，保留 venue、venueMarketId、原始标题、原始描述、原始状态、原始链接。
- 生成标准化 `TradeableInstrument`，每个标的绑定唯一平台、链、资产、marketId、outcomeId 和订单接口。
- 对异常响应、频控、超时进行重试、熔断和记录。

验收：

- 后台可看到两个平台的原始市场列表。
- 每个前台可交易标的都能追溯到唯一底层平台市场。
- 同步失败不会影响另一个平台。
- 低流动性、不可交易、结算规则不清或 API 异常标的不会进入推荐池。

#### 3.1.2 事件主题归组

能力：

- 使用 AI/文本相似度将 Polymarket 和 Predict.fun 的相似主题归入 `EventGroup`。
- `EventGroup` 只用于展示、搜索、相似推荐和持仓归组，不代表盘口合并。
- 同一主题下的多个 `TradeableInstrument` 仍然独立展示价格、链、资产、结算规则和交易入口。
- 低置信度、时间/规则差异明显或结算边界不一致的归组进入人工审核。

后台审核字段：

- 两个平台原始标题、描述、结束时间、结算规则、URL。
- AI 归组理由、置信度、差异点。
- 审核操作：确认归组、拒绝归组、创建独立主题、暂停展示。

验收：

- Polymarket 事件 A 与 Predict.fun 事件 B 是同一主题时，可归组展示。
- A/B 仍是两个独立可交易标的，不合并盘口、不合并深度、不合并成本池。
- 审核员拒绝归组后，两个标的不再作为同主题展示。

#### 3.1.3 标的列表与详情页

列表能力：

- 展示精选标的、热门标的、AI 推荐标的。
- 支持分类、搜索、热门、流动性、即将结束排序。
- 对同主题标的进行轻量归组或相似推荐。
- 不突出平台品牌，但保留必要的链/资产提示。

详情能力：

- 展示标的标题、结算规则、到期时间、当前价格/概率、流动性、成交量。
- 展示该标的自身盘口和成交历史。
- 展示快捷交易面板和专业交易面板切换。
- 展示同主题其他平台标的，但明确是独立交易入口。

同主题展示规则：

- 情况一：主题相同但投资参数、赔率、到期时间、结算边界或结果定义不同，只能做归组或相似推荐，前台仍展示为多个独立可交易标的。
- 情况二：主题相同且核心投资参数完全吻合，可以保留多个独立标的卡片，也可以在展示层合并成一个主题事件，并在详情页提供 Polymarket / Predict.fun 平台或标的切换。
- 合并主题只改变展示和选择路径，不改变交易模型；用户下单前必须落到一个具体 `TradeableInstrument`，并使用该标的绑定的平台、链、资产和订单接口。
- 若同主题跨平台出现套利空间，例如最优 Yes 价格 + 最优 No 价格小于 1.00 美元，需要在标的卡片、主题详情页或专业面板展示“套利空间”标识或横幅。
- 套利提示只作为发现信号，一期不承诺自动组合下单、不承诺无风险收益，也不做跨平台拆单。

验收：

- 用户能从列表进入标的详情并完成下单入口。
- 页面不把两个平台盘口合并为一个深度。
- 平台品牌不被突出展示，但订单详情和签名流程可追溯底层平台。
- 完全相同投资参数的同主题标的即使采用合并主题展示，也能切换并选择具体平台标的下单。
- 出现套利空间时，页面能展示提示；价格失效、流动性不足或任一平台不可交易时，提示自动撤下。

#### 3.1.4 快捷交易面板

定位：

- 面向小白用户，是一期核心竞争力。
- 类似简版期权/二元期权/一键买币体验。

字段：

- 买 Yes / 买 No。
- 投入金额。
- 当前价格/概率。
- 预计获得份额。
- 若结果成立可获得金额。
- 最大亏损。
- 到期时间。
- 结算条件入口。
- 授权状态。
- 钱包网络状态。
- 签名状态。
- 确认按钮。

文案规范：

- 使用用户能理解的收益表达，例如：“0.88 USDC 买入，若结果成立可变成 1 USDC”。
- Polymarket 标的优先使用 USDC，Predict.fun 标的优先使用 USDT。
- 如果前台统一写 USD，必须在详情或确认页说明实际结算资产。
- 必须展示最大亏损和结算条件入口，避免只强调收益。

验收：

- 小白用户可以不看盘口完成一次买入。
- 钱包未连接、网络不对、余额不足、授权不足、quote 过期时不能提交订单。
- 确认前展示最终金额、资产、网络、最大亏损和到期时间。

#### 3.1.5 专业交易面板

定位：

- 面向熟练用户。
- 支持更细的价格、数量和订单管理。

字段：

- Market / Limit。
- 买入 / 卖出。
- outcome。
- 价格。
- 数量。
- 金额。
- 当前标的盘口。
- 成交历史。
- 订单有效期。
- 最大滑点。
- 撤单入口。
- 底层平台/链弱展示。

验收：

- 用户可以创建 Market/Taker 订单。
- 用户可以创建 Limit/Maker 订单。
- 用户可以查看当前委托并撤单。
- 所有订单只提交到当前标的绑定的平台。

#### 3.1.6 Taker 下单

能力：

- 用户在当前 `TradeableInstrument` 上发起 Market 买入/卖出。
- 系统读取该标的绑定平台的最新 quote。
- 系统检查钱包连接、网络、余额、授权、滑点、quote 有效期。
- 用户完成必要签名后，系统通过底层平台 API 提交订单。
- 返回聚合平台订单号和底层平台订单映射。

验收：

- 点击 Polymarket 标的 A，只调用 Polymarket/Polygon 交易链路。
- 点击 Predict.fun 标的 B，只调用 Predict.fun/BSC 交易链路。
- 不做跨平台拆单，不自动选择其他平台。
- 下单失败时展示明确原因。

#### 3.1.7 Maker 挂单与撤单

能力：

- 用户在当前 `TradeableInstrument` 上输入限价、数量、有效期，创建 Maker 订单。
- 订单提交到该标的绑定平台。
- 支持查看当前委托、部分成交、完全成交、失败、撤单。
- 撤单只撤当前底层平台订单。

验收：

- Maker 不做跨平台复制挂单。
- Maker 不根据流动性自动切到另一个平台。
- 用户撤单成功后，底层平台订单也同步为 cancelled。
- 撤单失败时给出原因并允许重试。

#### 3.1.8 Web3 钱包与网络

能力：

- 支持 WalletConnect 类 EVM 钱包连接。
- EVM 地址作为用户唯一 ID。
- 记录用户连接钱包、支持链、当前链、授权状态。
- Polymarket 标的下单前自动请求切 Polygon。
- Predict.fun 标的下单前自动请求切 BSC/BNB Chain。
- 支持检测未添加链、切链失败、资产不足、授权不足、签名失败。

验收：

- 未连接钱包时不能下单。
- 网络不匹配时先触发切链。
- 切链失败时阻止下单。
- 余额和授权不足时提示用户补充或授权。
- 平台不创建托管账户，不保存用户私钥，不托管用户资产。

#### 3.1.9 订单、成交、持仓

能力：

- 以 EVM 地址查询和归集用户订单、成交、持仓。
- 保留聚合平台订单号。
- 每笔聚合订单只映射一个底层平台订单。
- 持仓可以按 `EventGroup` 做同主题归组展示，但不能合并为同一成本池。
- 持仓必须展示来源标的、底层平台、资产、链、份额、平均成本、当前价格、结算状态。

验收：

- 用户能看到自己在两个平台的订单和持仓。
- 同主题持仓可以归组，但每个持仓来源清晰。
- 订单详情可展开查看底层平台、链、订单 hash、签名 hash、失败原因。

#### 3.1.10 AI 自动报单

能力：

- 采集上手平台热门、高流动性、赔率清晰、规则明确的标的。
- 根据热度、流动性、价格/概率、成交量、到期时间、规则清晰度生成候选。
- 输出自动报单机会 `AutoOrderOpportunity`。
- 为推荐生成快捷文案、banner、快捷下单入口。
- 支持运营审核上线、下架、排序、风险暂停。

筛选规则：

- 必须是 API 可交易标的。
- 必须有足够流动性和成交活跃度。
- 必须结算规则清晰。
- 必须未进入异常临近结算窗口。
- 必须 quote 可用且未 stale。
- 低流动性、规则复杂、状态异常、API 异常标的不进入推荐。

下架规则：

- quote 过期或盘口 stale。
- 流动性下降到阈值以下。
- 事件临近结算或状态异常。
- 平台 API 异常。
- 运营手动下架。
- 风控命中。

验收：

- 自动报单 banner 能从热门高流动性标的生成快捷购买入口。
- 推荐文案同时展示概率、到期时间、最大亏损、结算条件入口。
- 推荐标的异常时自动下架，不允许继续快捷下单。

#### 3.1.11 风控与合规

能力：

- 地区限制：敏感地区 IP 禁止交易。
- 标的门禁：不可交易、规则不清、API 异常、临近异常结算标的不允许下单。
- 价格保护：quote 有效期、最大滑点、盘口 stale 检测。
- 钱包保护：网络不匹配、余额不足、授权不足、签名失败时阻止提交。
- 平台熔断：第三方 API 异常、订单失败率过高、撤单失败率过高时暂停对应平台交易。
- 推荐风控：AI 自动报单标的异常时自动下架。
- 审计日志：记录 EVM 地址、IP、地区判断、标的、venue、chainId、quote、签名、提交、成交、撤单、失败原因。

验收：

- 风控命中时不提交真实订单。
- 后台可查看每笔订单完整执行链路。
- 平台异常时前台显示“暂不可交易/报价更新中”。

### 3.2 P1 一期增强

- 快捷面板文案 A/B 测试。
- 首页自动报单 banner 多样式。
- 专业面板展示更完整盘口深度和成交历史。
- 同主题标的价格对比，但不合并深度。
- 同主题订单/持仓归组展示。
- AI 推荐后台：推荐理由、推荐分、文案审核、排序、下架原因。
- 通知：成交、部分成交、撤单、事件结算。

### 3.3 暂不进入一期

- 跨平台盘口深度合并。
- 聚合订单簿。
- 跨平台最优路由。
- 大额拆单。
- 同一 Maker 订单多平台复制挂单。
- 自动选择其他平台执行。
- 托管账户 / managed balance。
- 帮用户开设底层平台账户。
- 自动跨链换币 / rebalance。
- 代付 Gas。
- 自研预测市场。
- 自研 Oracle。
- 无风险套利 MM 产品化。
- 对外开放成交 API。
- 暗池撮合。

## 4. 核心页面

### 4.1 首页 / 标的列表

模块：

- 搜索框。
- 分类筛选。
- 热门标的。
- AI 自动报单 banner。
- 同主题归组。
- 当前价格/概率。
- 流动性/成交量。
- 到期时间。
- 标的状态。

关键交互：

- 点击推荐 banner 进入快捷交易。
- 点击标的进入详情。
- 同主题下展示多个独立可交易标的。
- 不可交易标的不显示下单按钮。

### 4.2 标的详情页

模块：

- 标题、描述、结算规则、到期时间。
- 当前价格/概率。
- 流动性、成交量、价格走势。
- 当前标的盘口。
- 快捷交易面板。
- 专业交易面板。
- 同主题其他标的。
- 当前用户持仓摘要。
- 当同主题标的核心投资参数完全一致时，可展示平台/标的切换控件。
- 当同主题跨平台最优 Yes + 最优 No 小于 1.00 美元且通过有效性检查时，展示套利空间标识或横幅。

关键交互：

- 快捷/专业模式切换。
- 查看结算规则。
- 查看同主题其他标的。

### 4.3 快捷交易面板

- 快捷面板重点让用户快速理解投入金额、买入结果、最大亏损和成立后可获得金额。

模块：

## 9. 验收用例

### 9.1 事件归组

- Polymarket 事件 A 与 Predict.fun 事件 B 是同一主题时，前台可归组展示。
- 当 A/B 投资参数、赔率、到期时间或结算边界不同时，A/B 仍然是两个独立可交易标的。
- 当 A/B 核心投资参数完全一致时，前台可以合并展示为一个主题事件，但详情页必须提供 Polymarket / Predict.fun 或具体标的切换。
- A/B 不合并盘口深度，不合并成本池。
- 用户点击 A 下单只调用 A 绑定平台；点击 B 下单只调用 B 绑定平台。
- 用户在合并主题详情页下单前，必须已经选中一个具体平台标的。

### 9.2 快捷交易

- 自动报单 banner 展示推荐文案、概率、到期时间、最大亏损、结算条件入口。
- 用户输入金额后看到预计获得和成功后收益。
- quote 过期时确认按钮失效。
- 钱包未连接、网络不匹配、余额不足、授权不足时无法提交。

### 9.3 专业交易

- 用户可以选择 Market/Limit。
- 用户可以输入价格、数量、有效期。
- 用户可以查看当前标的盘口和成交历史。
- 用户可以创建 Maker 挂单并撤单。

### 9.4 单平台执行

- Maker 挂单在当前标的对应平台提交。
- Taker 下单在当前标的对应平台提交。
- 撤单只撤对应底层订单。
- 不做拆单、不复制挂单、不自动选择其他平台。

### 9.5 钱包网络

- Polymarket 标的下单前切到 Polygon。
- Predict.fun 标的下单前切到 BSC。
- 用户拒绝切链时订单不提交。
- 用户未添加链时提示添加或切换失败原因。

### 9.6 订单持仓

- 用户能看到两个平台的订单和持仓。
- 同主题持仓可以归组展示。
- 同主题持仓不合并为同一成本池。
- 订单详情可展开查看底层平台、链、资产、订单 hash。

### 9.7 自动报单下架

- quote stale 时自动下架。
- 流动性下降到阈值以下时自动下架。
- 事件临近异常结算窗口时自动下架。
- 平台 API 异常时自动下架。
- 运营可手动下架。

### 9.8 套利空间提示

- 当同主题跨平台最优 Yes + 最优 No 小于 1.00 美元，且扣除手续费、滑点、链上成本后仍可能成立时，标的卡片或详情页展示套利空间提示。
- 套利提示必须展示涉及的两个具体标的、方向、价格和有效时间。
- 任一价格过期、盘口不可交易、流动性不足、平台 API 异常或网络不支持时，套利提示自动消失。
- 一期套利提示只做信息展示，不自动组合下单、不做拆单、不承诺无风险收益。

### 9.9 风控

- 敏感地区 IP 用户无法交易。
- 不可交易标的无下单入口。
- 平台 API 异常时，对应平台标的不可交易。
- 所有下单、撤单、失败都有审计日志。

## 10. 待确认问题

- 一期主网交易是否先做白名单灰度。
- 一期是否必须支持卖出/平仓，还是先支持买入、挂单、撤单。
- WalletConnect、GemW、Passkey 钱包的优先级。
- Polymarket 和 Predict.fun API Key、测试环境、签名凭证是否已准备。
- 自动报单推荐阈值：最低流动性、最低成交量、到期时间范围、最低赔率可读性。
- 自动报单文案是否需要运营审核后才能上线。
- 地区限制名单和合规文案由谁确认。
